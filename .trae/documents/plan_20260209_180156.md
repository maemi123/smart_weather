# 修改计划：修复风速效果显示异常并调整阈值

用户反馈“大风效果”显示为方框，且触发过于频繁。我们需要修复符号兼容性问题，并提高风速触发阈值。

## 1. 修复符号显示异常（方框问题）

*   **原因分析**：
    *   目前使用的符号 `≡` (Triple Bar, U+2261) 在某些字体（包括 Segoe UI Emoji）下可能无法正常渲染或显示样式不符预期。
    *   Emoji `💨` (Dash Symbol, U+1F4A8) 在 Windows 下依赖 Segoe UI Emoji，但如果字体回退机制失效，也可能显示异常。

*   **解决方案**：
    *   弃用可能存在兼容性问题的文本符号。
    *   **改用 Matplotlib 原生绘图**：不再使用 `ax.text` 绘制文本符号，而是使用 `ax.plot` 绘制简单的线条或箭头来表示风。
    *   **设计方案**：在风速较大的高度层，绘制几条水平的**流线 (Streamlines)** 或 **波浪线**。
        *   使用 `marker='$'` 或自定义路径可能太复杂，简单的**虚线**或**箭头**效果最好且兼容性 100%。
        *   或者使用 ASCII 字符 `~` (波浪号) 或 `>>>` (大于号)，这些在任何字体下都能正常显示。

    *   **最终决定**：使用 ASCII 字符 **`>>>`** (表示强劲气流) 或 **`~~~~`** (表示气流扰动)，配合不同的颜色和透明度。这两个符号在任何字体下都不会变成方框。

## 2. 调整风速触发阈值

*   **用户需求**：将触发阈值从 `18 m/s` 提高到 **`25 m/s`**。
*   **调整逻辑**：
    *   **阈值 1**: `> 25 m/s` (约 10 级风，强风)。显示橙色流线。
    *   **阈值 2**: `> 40 m/s` (约 13 级风，急流中心)。显示红色流线。

## 执行步骤

1.  **修改 `sounding_plotter.py`**:
    *   定位到 `_plot_simple` 方法中的大风可视化部分。
    *   修改触发条件：`if wind_spd > 25:`。
    *   替换绘图逻辑：
        *   弃用 `wind_icon` 和 `fontname='Segoe UI Emoji'`。
        *   改为绘制 ASCII 字符 `~` 或 `>`。
        *   例如：`wind_marker = ">>>" if wind_spd > 40 else ">>"`。
        *   使用标准字体（如 Arial 或默认 sans-serif），不再强制指定 Emoji 字体，避免方框。

2.  **验证**：
    *   确保修改后的代码不再产生 Unicode 相关错误。
    *   确保逻辑阈值符合用户要求的 25m/s。

## 代码变更预览

```python
# 原代码
if wind_spd > 18:
    wind_icon = "💨" if wind_spd > 30 else "≡"
    # ... 使用 Segoe UI Emoji 绘制

# 新代码
if wind_spd > 25:
    # 使用 ASCII 字符，100% 兼容
    # 25-40m/s: 橙色 ">>"
    # >40m/s: 红色 ">>>"
    wind_marker = ">>>" if wind_spd > 40 else ">>" 
    wind_color = "#e74c3c" if wind_spd > 40 else "#f39c12"
    
    # 使用默认字体绘制，加粗
    ax.text(0.55, y_pos, wind_marker, fontsize=16, color=wind_color, 
            ha='center', va='center', alpha=alpha, fontweight='bold') 
```
