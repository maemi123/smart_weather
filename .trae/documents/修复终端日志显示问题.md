# 修复终端日志显示问题

## 问题描述
页面能正常加载，但终端不显示 Flask 的日志信息（如请求日志、print输出等）。

## 根本原因
在 `app.py` 第30-33行，为了修复Windows控制台UTF-8编码问题，代码重新包装了 `sys.stdout` 和 `sys.stderr`：

```python
# 修复Windows控制台编码问题
if sys.platform == 'win32':
    sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8')
    sys.stderr = io.TextIOWrapper(sys.stderr.buffer, encoding='utf-8')
```

这会导致：
1. `print()` 输出的内容被缓冲，不立即显示在终端
2. Flask/Werkzeug 的日志输出可能无法正常显示
3. 输出流被重新包装后，与Flask的日志机制产生冲突

## 解决方案

### 推荐方案：添加 `line_buffering=True`
修改编码修复代码，添加行缓冲参数，确保输出立即刷新：

```python
# 修复Windows控制台编码问题
if sys.platform == 'win32':
    sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8', line_buffering=True)
    sys.stderr = io.TextIOWrapper(sys.stderr.buffer, encoding='utf-8', line_buffering=True)
```

### 备选方案：只在必要时修改
检查当前编码是否已经是UTF-8，避免不必要的重新包装：

```python
# 修复Windows控制台编码问题
if sys.platform == 'win32':
    import locale
    if sys.stdout.encoding != 'utf-8':
        sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8', line_buffering=True)
    if sys.stderr.encoding != 'utf-8':
        sys.stderr = io.TextIOWrapper(sys.stderr.buffer, encoding='utf-8', line_buffering=True)
```

## 实施步骤
1. 修改 `app.py` 第31-32行，添加 `line_buffering=True` 参数
2. 重启 Flask 服务
3. 验证终端是否能正常显示日志

## 验证方法
1. 访问任意页面（如 `/history/trend`）
2. 观察终端是否显示类似以下的请求日志：
   ```
   127.0.0.1 - - [17/Feb/2026 10:00:00] "GET /history/trend HTTP/1.1" 200 -
   ```
3. 观察 `print()` 输出是否能正常显示
