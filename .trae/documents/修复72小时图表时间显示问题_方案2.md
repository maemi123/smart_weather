# 修复72小时精细化图表时间显示问题（方案2）

## 问题确认

两个图表（72小时精细化和多模式对比）使用的是**同一个后端数据处理函数** `_process_hourly_data`，所以数据本身没有问题。问题出在**前端图表的配置差异**。


## 问题根源

1. **X轴类型问题**：72小时图表使用 `type: 'category'`，这会导致Plotly将X轴视为分类数据，可能造成时间显示不准确。

2. **标签显示问题**：72小时图表使用 `dayLabels`（如"02月17日"）作为刻度标签，丢失了具体时间信息。

3. **数据点对应问题**：使用 `category` 类型时，Plotly可能无法正确对齐数据点和时间标签。

## 解决方案

按照多模式对比图表的方式修改72小时图表：

### 修改1：移除 `type: 'category'`

```javascript
xaxis: {
    title: '日期时间',
    // type: 'category',  // 移除这行
    tickangle: 45,
    gridcolor: '#f0f0f0',
    tickmode: 'array',
    tickvals: labels.filter((_, i) => i % 12 === 0),
    ticktext: labels.filter((_, i) => i % 12 === 0)  // 使用labels而不是dayLabels
}
```

### 修改2：改进标签显示（可选，更美观）

按照多模式对比的方式，创建更友好的时间标签：

```javascript
// 在 updateDetailedChart 函数中
const labels = data.timestamps ? data.timestamps.slice(0, maxPoints) : data.full_labels.slice(0, maxPoints);

// 创建显示标签（每24小时显示日期，其他显示时间）
const displayLabels = labels.map((label, index) => {
    if (index % 24 === 0) {
        // 每24小时显示完整日期时间
        return label;  // 如 "02-17 14:00"
    } else if (index % 6 === 0) {
        // 每6小时显示时间
        return label.split(' ')[1];  // 如 "14:00"
    }
    return '';  // 其他不显示
});

// 在layout中使用
xaxis: {
    title: '日期时间',
    tickangle: 45,
    gridcolor: '#f0f0f0',
    tickmode: 'array',
    tickvals: labels.filter((_, i) => i % 6 === 0),  // 每6小时一个刻度
    ticktext: displayLabels.filter((_, i) => i % 6 === 0)
}
```
