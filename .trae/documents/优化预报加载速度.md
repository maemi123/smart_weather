# 优化多模式预报页面加载速度

## 问题分析

### 当前问题
`/advanced-forecast` 页面加载慢的原因：

1. **串行API请求**: 在 `fetch_multi_model_forecast` 中，对每个模型（best_match, ecmwf_ifs, gfs_seamless, icon_global）都是串行请求
2. **多个API调用**: 
   - 72小时精细化预报：1个API调用
   - 多模式预报：4个模型 = 4个API调用
   - 总共：5个API调用，每个 timeout=25秒
3. **无缓存机制**: 每次访问页面都重新请求API

### 时间估算
- 单个API请求：约2-5秒（正常情况）
- 5个串行请求：约10-25秒
- 如果网络慢：可能超过1分钟

## 解决方案

### 方案1: 并发请求（推荐）
使用 `concurrent.futures.ThreadPoolExecutor` 并发请求多个模型

**优点**: 
- 大幅减少总请求时间（从串行变并行）
- 5个请求只需约3-5秒

**修改文件**: `advanced_forecast_service.py`

### 方案2: 添加缓存机制
使用 Flask-Caching 或简单的内存缓存，缓存API响应

**优点**:
- 重复访问时几乎瞬间加载
- 减少API调用次数

**修改文件**: `advanced_forecast_service.py`, `app.py`

### 方案3: 前端异步加载
页面先渲染框架，数据通过AJAX异步加载

**优点**:
- 用户感知更快
- 可以显示加载进度

**修改文件**: `templates/advanced_forecast.html`, `app.py`

## 推荐实施

**优先级**: 方案1 > 方案2 > 方案3

        pass
    
    with ThreadPoolExecutor(max_workers=4) as executor:
        futures = {executor.submit(fetch_single_model, m): m for m in target_models}
        for future in as_completed(futures):
            model_name = futures[future]
            try:
                result = future.result()
                if result:
                    results[model_name] = result
            except Exception as e:

    return results
