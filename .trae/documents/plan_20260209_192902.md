# 修复农业气象总览页数据越界错误

## 问题分析
用户访问 `/agro-dashboard` 时遇到 `IndexError: list index out of range`。
错误发生在 `daily_map[date_str]['temps'].append(temps[i])` 这一行。

**根本原因**：`timestamps`（时间戳列表）的长度大于 `temps`（温度列表）或 `precips`（降水列表）的长度。
在遍历 `timestamps` 时，如果索引 `i` 超过了 `temps` 或 `precips` 的最大索引，就会触发此错误。这通常发生在 API 返回的数据不完整或解析过程中长度不一致时。

## 修复计划

1.  **修改 `app.py`**：
    *   定位到 `agro_dashboard` 路由。
    *   在进入循环前，计算三个列表（`timestamps`, `temps`, `precips`）的**最小长度**。
    *   使用这个最小长度作为循环的范围，确保 `i` 永远不会越界。

## 执行步骤

1.  读取 `app.py`。
2.  修改数据聚合循环的逻辑。

```python
# 修改前
for i, ts in enumerate(timestamps):
    # ...

# 修改后
min_len = min(len(timestamps), len(temps), len(precips))
for i in range(min_len):
    ts = timestamps[i]
    # ...
```
