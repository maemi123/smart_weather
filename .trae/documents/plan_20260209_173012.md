# 修改计划：优化卡通直观图（原卡通简化版）

我们将对“卡通简化版”图表进行全面的视觉升级和 Bug 修复，使其更直观、美观且兼容性更好。

## 1. 核心绘图逻辑优化 (`sounding_plotter.py`)

### A. 修复高度比例与 0 度层显示

* **问题**：目前雪花可能显示在 0 度层下方，高度比例感观不对。

* **方案**：

  * 确保 Y 轴映射基准 `min_hght` 严格取自地面海拔（而非 0 米），避免地面层悬空。

  * 统一所有图层元素（线条、图标、文字）的 Y 坐标计算公式，确保相对位置绝对准确。

  * 调整 0 度层文字的垂直对齐方式，避免视觉误差。

### B. 增强天气现象可视化（雪花/水滴/大风）

* **多列图标**：不再单列显示。在检测到高湿度层时，将在 X 轴方向（0.1 到 0.9 范围）随机散布 3-6 个雪花或水滴图标，形成“层”的视觉效果。

* **大风效果**：遍历高度层，若某层风速超过 18m/s（8级风），在背景绘制流线型符号（如 `💨` 或 `≡`），风速越大符号越密集。

### C. 修复 Emoji 显示为方框（关键修复）

* **问题**：Windows 下中文字体不包含彩色 Emoji，导致“云层较厚”、“关键分界线”等图标显示异常。

* **方案**：采用**图文分离绘制**策略。

  * 将 `☁️ 云层较厚` 拆分为 `☁️` 和  ` 云层较厚`。

  * `☁️`：强制使用 `Segoe UI Emoji` 字体。

  * &#x20;` 云层较厚`：使用中文字体（如 `Microsoft YaHei`）。

  * 对“关键分界线”、“交通堵塞”等所有含 Emoji 的标签应用此策略。
