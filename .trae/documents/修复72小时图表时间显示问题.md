# 修复72小时精细化图表时间显示问题

## 问题描述

在 `/advanced-forecast` 页面的72小时精细化图表中，鼠标悬停在温度峰值处显示的时间都在10:00左右，而正常情况下午温度峰值应该在14:00左右。

## 根本原因分析

### 1. 数据时间处理对比

**72小时图表（有问题）** - [advanced\_forecast\_service.py 第195-198行](file:///d:/pythonProject/smart_weather/advanced_forecast_service.py#L195-L198)：

```python
# 直接使用API返回的时间（已经是北京时间）
dt = datetime.fromisoformat(t.replace('Z', '+00:00'))
# 修复：不进行+8转换
# dt = dt + timedelta(hours=8)  # 删除这行
```

这里虽然注释说"API返回的已经是北京时间"，但实际上 Open-Meteo API 返回的是 **UTC时间**（带Z后缀），需要转换为北京时间（UTC+8）。

**多模式对比图表（正常）** - 使用的是同样的数据源，但图表渲染方式不同。

### 2. 图表X轴配置对比

**72小时图表（有问题）** - [advanced\_forecast.html 第544-552行](file:///d:/pythonProject/smart_weather/templates/advanced_forecast.html#L544-L552)：

```javascript
xaxis: {
    title: '日期时间',
    type: 'category',  // ← 使用category类型
    tickangle: 45,
    gridcolor: '#f0f0f0',
    tickmode: 'array',
    tickvals: labels.filter((_, i) => i % 12 === 0),
    ticktext: dayLabels.filter((_, i) => i % 12 === 0)
}
```

**多模式对比图表（正常）** - [advanced\_forecast.html 第791-798行](file:///d:/pythonProject/smart_weather/templates/advanced_forecast.html#L791-L798)：

```javascript
xaxis: {
    title: '时间',
    gridcolor: '#f0f0f0',
    tickangle: 45,
    tickmode: 'array',
    tickvals: timeLabels.filter((_, i) => i % 12 === 0),
    ticktext: displayLabels.filter((_, i) => i % 12 === 0),
    showgrid: true
    // 注意：没有指定 type: 'category'，默认是 linear/auto
}
```

### 3. 问题总结

主要问题有两个：

1. **时间转换问题**：Open-Meteo API 返回的是 UTC 时间（如 `2026-02-17T06:00:00Z`），代码中虽然注释说"已经是北京时间"，但实际上需要转换为 UTC+8 才是正确的北京时间。

2. **X轴类型问题**：72小时图表使用了 `type: 'category'`，这会导致 Plotly 将X轴视为分类数据而不是时间数据，可能造成时间显示不准确。

## 解决方案

### 方案1：修复时间转换（推荐）

在 [advanced\_forecast\_service.py](file:///d:/pythonProject/smart_weather/advanced_forecast_service.py#L193-L210) 中恢复正确的UTC到北京时间转换：

```python
for i, t in enumerate(times):
    try:
        # Open-Meteo API返回的是UTC时间（带Z后缀），需要转换为北京时间（UTC+8）
        dt = datetime.fromisoformat(t.replace('Z', '+00:00'))
        dt = dt + timedelta(hours=8)  # UTC+8 = 北京时间
        
        # 如果是详细模式且指定了current_hour
        if current_hour is not None and detailed:
            # 只跳过明显过去的时间（比如超过3小时前）
            if dt.hour < current_hour - 3 and dt.date() == datetime.now().date():
                continue

        bj_times.append(dt.strftime("%Y-%m-%d %H:%M"))
        full_labels.append(dt.strftime("%m/%d %H:%M"))
        date_labels.append(dt.strftime("%m/%d"))
        hour_labels.append(f"{dt.hour:02d}:00")
        day_labels.append(dt.strftime("%m月%d日"))
    except Exception as e:
        print(f"时间解析错误: {t}, 错误: {e}")
        continue
```

### 方案2：修改X轴类型（备选）

在 [advanced\_forecast.html](file:///d:/pythonProject/smart_weather/templates/advanced_forecast.html#L544-L552) 中移除 `type: 'category'`：

```javascript
xaxis: {
    title: '日期时间',
    // type: 'category',  // 移除这行，让Plotly自动检测
    tickangle: 45,
    gridcolor: '#f0f0f0',
    tickmode: 'array',
    tickvals: labels.filter((_, i) => i % 12 === 0),
    ticktext: dayLabels.filter((_, i) => i % 12 === 0)
}
```

## 推荐方案

**采用方案1（修复时间转换）**，因为这是根本问题。Open-Meteo API 明确返回 UTC 时间，必须转换为北京时间才能正确显示。

## 验证方法

1. 修复后访问 `/advanced-forecast` 页面
2. 查看72小时精细化图表
3. 鼠标悬停在温度峰值处，应该显示14:00左右（下午温度最高）
4. 鼠标悬停在温度谷值处，应该显示05:00-07:00左右（凌晨温度最低）
