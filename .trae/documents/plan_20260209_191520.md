# 修复 `agro_calculator.py` 返回值解包错误

## 问题分析
用户在访问 `/agro-dashboard` 时遇到 `TypeError: cannot unpack non-iterable int object` 错误。

根据 Traceback：
```python
File "D:\pythonProject\smart_weather\app.py", line 1121, in agro_dashboard
    score, deductions = agro_calculator.calculate_suitability_score(stage, today_weather)
```

这说明 `calculate_suitability_score` 方法返回的是一个整数（`int`），而不是期望的元组（`tuple`），导致无法解包为 `score` 和 `deductions`。

虽然之前的搜索结果显示代码逻辑是返回元组 `return max(0, score), deductions`，但实际运行中可能存在某种边界情况导致它只返回了 `0` 或其他整数。

**最可能的原因**：
在 `calculate_suitability_score` 函数的开头有参数检查：
```python
if not crop_stage_info or not current_weather:
    return 0
```
如果 `crop_stage_info`（作物阶段信息）或 `current_weather`（天气数据）为空，它直接返回了整数 `0`，而不是 `(0, [])`。这与调用处的解包逻辑不兼容。

## 修复计划

1.  **修改 `agro_calculator.py`**：
    *   定位到 `calculate_suitability_score` 方法。
    *   修改早停（early return）逻辑，使其返回元组 `(0, [])` 而不是单值 `0`。

## 执行步骤

1.  读取 `agro_calculator.py` 确认问题代码。
2.  修改返回语句。
3.  （可选）检查 `app.py` 中的调用逻辑，确保传入的数据不为空，但这属于防御性编程，核心修复在于被调用函数。

```python
# 修改前
if not crop_stage_info or not current_weather:
    return 0

# 修改后
if not crop_stage_info or not current_weather:
    return 0, ["数据缺失无法计算"]
```
