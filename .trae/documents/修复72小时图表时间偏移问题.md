# 修复72小时图表时间偏移问题

## 问题分析

**现象**: 72小时精细化图表时间显示慢了6小时（之前是7小时、8小时）

**根本原因**: 在 `_process_hourly_data` 函数中，`current_hour` 参数的处理逻辑有问题。

### 问题代码 (advanced_forecast_service.py 第200-204行)

```python
# 如果是详细模式且指定了current_hour
if current_hour is not None and detailed:
    # 只跳过明显过去的时间（比如超过3小时前）
    if dt.hour < current_hour - 3 and dt.date() == datetime.now().date():
        continue
```

**问题说明**:
1. 这个逻辑只比较小时数，没有考虑分钟
2. 条件 `dt.date() == datetime.now().date()` 可能因为时区问题导致判断错误
3. 跳过数据的逻辑不正确，导致时间偏移

### 对比多模式对比图表

多模式对比图表调用 `fetch_multi_model_forecast()` 时**不传递** `current_hour` 参数，所以不会触发这个有问题的逻辑，时间显示正常。

## 解决方案

### 方案1: 移除跳过过去时间的逻辑（推荐）

由于API返回的数据本身就是从当前小时开始的预报数据，不需要额外跳过任何数据。

**修改内容**:
```python
# 删除或注释掉这段代码
# if current_hour is not None and detailed:
#     if dt.hour < current_hour - 3 and dt.date() == datetime.now().date():
#         continue
```

### 方案2: 修复跳过逻辑

如果确实需要跳过过去的时间，应该使用更精确的比较：

```python
if current_hour is not None and detailed:
    now = datetime.now()
    # 只跳过真正过去的时间点
    if dt < now - timedelta(hours=1):
        continue
```

1. 修改 `advanced_forecast_service.py` 中的 `_process_hourly_data` 函数
2. 移除或修复 `current_hour` 相关的跳过逻辑
3. 验证72小时图表时间显示是否正确
