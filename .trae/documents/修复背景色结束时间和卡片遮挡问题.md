# 修复背景色结束时间和卡片遮挡问题

## 问题分析

### 问题1: 灰色背景结束在23:00而不是0:00

**原因分析：**
当前代码中，每一天的背景色范围是通过遍历 `labels` 数组来确定的：
- 当检测到日期变化时，前一天的结束点被设置为 `labels[index - 1]`
- 这意味着如果新的一天从 "02-19 00:00" 开始，前一天的结束点就是 "02-18 23:00"

**解决方案：**
需要修改逻辑，让每一天的背景色延伸到当天的最后一个小时（23:00），而不是下一天的开始时间减一。具体来说：
- 记录每一天的开始时间
- 当检测到日期变化时，前一天的结束应该是前一天的23:00
- 或者更简单的方式：使用当天的最后一个数据点作为结束

### 问题2: 温度极值、累计降水等卡片被图表挡住

**原因分析：**
卡片的 CSS 样式可能存在 z-index 或 margin 问题，导致被图表容器遮挡。

**解决方案：**
- 增加卡片容器的 margin-top
- 或者确保卡片容器的 z-index 高于图表

## 修复方案

### 修复1: 修改背景色范围逻辑

在 `updateDetailedChart` 函数中，修改 `dayRanges` 的生成逻辑：

```javascript
// 识别每天的边界（用于背景色交替）
const dayRanges = [];
let currentDate = '';
let dayStartIndex = 0;

labels.forEach((label, index) => {
    const date = label.split(' ')[0];
    if (date !== currentDate) {
        if (currentDate !== '') {
            // 保存前一天的结束点（使用当天的最后一个数据点）
            dayRanges[dayRanges.length - 1].end = labels[index - 1];
        }
        // 开始新的一天
        dayStartIndex = index;
        dayRanges.push({
            start: label,
            end: '',
            index: dayRanges.length
        });
        currentDate = date;
    }
});
// 填充最后一天的结束点
if (dayRanges.length > 0) {
    dayRanges[dayRanges.length - 1].end = labels[labels.length - 1];
}
```

但是这样还是会在23:00结束。要让它延伸到0:00，需要：
- 对于非最后一天，结束点应该是下一天的00:00
- 或者使用一个更宽的范围

更好的方案是：
```javascript
// 识别每天的边界（用于背景色交替）
const dayRanges = [];
let currentDate = '';

labels.forEach((label, index) => {
    const date = label.split(' ')[0];
    if (date !== currentDate) {
        // 开始新的一天
        dayRanges.push({
            start: label,
            end: '',
            date: date
        });
        currentDate = date;
    }
    // 更新当前天的结束点（总是更新到当前遍历到的点）
    if (dayRanges.length > 0) {
        dayRanges[dayRanges.length - 1].end = label;
    }
});
```

### 修复2: 增加卡片容器的上边距

在卡片容器（grid布局的div）上增加 `margin-top`：

```html
<div style="display: grid; grid-template-columns: repeat(auto-fit, minmax(200px, 1fr)); gap: 15px; margin-top: 40px;">
```

将 `margin-top: 20px` 改为 `margin-top: 40px` 或更大。
