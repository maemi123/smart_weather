### 1. Fix Missing Indices (CAPE as "N/A")
The issue is that `SoundingDataParser` relies on regex to extract indices from the raw text report. However, the University of Wyoming website often puts these indices in a separate `<PRE>` block *after* the data table, and the current regex might be missing them if the format varies slightly or if the parser stops reading too early.

**Plan:**
*   Modify `_parse_indices` in `sounding_parser.py` to be more robust.
*   **Fallback Calculation**: If indices like CAPE/CIN are missing from the raw text but profile data exists, use **MetPy** to calculate them in `SoundingAnalyzer`. This ensures we always have critical parameters.

### 2. Fix Historical Data Fetching Error
The error "Failed to fetch data" when selecting a history date suggests an issue with how the date is passed or processed.
*   The frontend sends `date=YYYY-MM-DD HH:MM`.
*   The backend `get_upperair_data` parses this correctly.
*   The issue likely lies in `SoundingDataParser.fetch_sounding_data`. When a specific `target_time` is provided, it requests that exact time. If Wyoming doesn't have data for *exactly* that hour (e.g., user picks 14:00 but data is at 12:00), it fails.

**Plan:**
*   Modify `fetch_sounding_data` or the route handler to snap the requested time to the nearest 00Z or 12Z, as sounding data is only available twice a day.
*   Update `get_latest_valid_time` to be more flexible or ensure the frontend sends standard times.
