# 随机森林误差校正训练脚本计划

## 目标

创建一个完整的随机森林误差校正训练脚本，对 ECMWF（通过 Open-Meteo 获取）的预报数据进行校正。

## 数据分析

### 预报数据结构

* **路径**: `data/openmeteo/HGH_forecast_2026-*.csv`

* **文件名格式**: `HGH_forecast_2026-02-15_20260215_041555.csv`

  * 第一个日期: 预报起始日期（数据的第一天）

  * 第二个时间戳: 文件生成时间（用于推断起报时间）

* **关键字段**:

  * `datetime_beijing`: 预报时间

  * `temperature_2m`: 温度预报

  * `relative_humidity_2m`: 湿度预报

  * `precipitation`: 降水量预报

  * `wind_speed_10m`: 风速预报

  * `wind_direction_10m`: 风向预报

  * `pressure_msl`: 气压预报

### 实况数据结构

* **文件**: `data/hangzhou_observed_20260111_20260221.csv`

* **时间范围**: 2026-01-11 00:00 到 2026-02-21 23:00

* **关键字段**:

  * `datetime`: 北京时间

  * `temp`: 实况温度

  * `rhum`: 实况湿度

  * `prcp`: 实况降水

  * `wspd`: 实况风速

  * `wdir`: 实况风向

  * `pres`: 实况气压

### 起报时间推断规则

根据文件生成时间推断 ECMWF 起报时次：

* **生成时间 < 14:30**: EC12Z（前一天 20:00 北京时间起报）

* **生成时间 >= 14:30**: EC00Z（当天 08:00 北京时间起报）

例如：

* `20260215_041555` → 04:15 生成 → EC12Z → 起报时间 = 2026-02-14 20:00

* `20260215_161550` → 16:15 生成 → EC00Z → 起报时间 = 2026-02-15 08:00

### 步骤2: 特征工程

**特征列表**:

1. **数值特征**（预报值本身）:

   * forecast\_temp: 温度预报

   * forecast\_rhum: 湿度预报

   * forecast\_wspd: 风速预报

   * forecast\_pres: 气压预报

2. **时间特征**:

   * hour: 小时 (0-23)

   * month: 月份 (1-12)

   * day\_of\_week: 星期几 (0-6)

   * is\_day: 是否白天 (6:00-18:00)

3. **预报时效特征**:

   * lead\_hours: 预报时效（小时）

   * lead\_category: 时效分类 (0-24h, 24-48h, 48-72h, 72-120h, 120-168h)

4. **衍生特征**:

   * is\_weekend: 是否周末

   * forecast\_hour\_sin/cos: 小时的周期性编码

   * season:季节

## 注意事项

1. **数据清洗**:

   * 移除实况缺失的样本

   * 移除预报时效超过 168 小时的数据（精度下降严重）

2. **特征工程**:

   * 对周期性特征使用 sin/cos 编码

   * 标准化数值特征（随机森林不需要，但记录均值方差供后续使用）

3. **模型调优**:

   * 初始使用默认参数

   * 后续可根据评估结果调整

4. **降水处理**:

   * 降水先暂不训练模型（需要分类模型或特殊处理）

   * 后续可添加降水概率校正
